BPS工作空间缺省用户权限配置说明

BPS Workspace 是Primeton BPS 产品提供的业务流程统一管理工作区。您可以使用由BPS 产品默认提供的组织机构模型的用户来进行业务流程的各种管理和优化工作（当然，实际项目中完全可以采用BPS的开放接口进行替换）。

BPS 默认提供的六种内置权限集：

  • bpsclient：客户端使用权限，给用户提供了启动流程、查看工作项、领取/执行工作项、代理设置等功能；
  • bpsadmin：受管引擎实例管理权限，给用户提供了引擎实例的注册、管理、配置等功能（该权限默认仅为sysadmin用户拥有）；
  • bpsmanager：流程管理监控权限，可以使用工作流操作平台，通过它业务流程管控人员不仅可以对流程实例进行管理，同时还可以对业务操作人员的流程动作进行监控；
  • bpscustom：业务流程配置权限，可以使用基于BPS流程引擎实现的Web端的流程业务化定制工具，通过它业务流程管理人员可以以业务视角进行流程的业务化建模、配置与调整；
  • bpsconfig：受管引擎配置管理权限，可以使用系统管理工具，通过它系统管理员可以对工作流引擎进行配置和管理；
  • bpsbizcat：业务目录管理权限，提供业务目录创建、维护、授权，以及业务流程、业务资源导入导出功能；

BPS 的提供的默认用户及权限分配：

tiger： 客户端使用权限、流程管理监控权限、业务流程配置权限、受管引擎配置管理权限、业务目录管理权限；
goose： 客户端使用权限、流程管理监控权限；
fish： 客户端使用权限、流程管理监控权限、业务流程配置权限、受管引擎配置管理权限；
snoppy： 客户端使用权限、受管引擎配置管理权限、业务目录管理权限；
micky： 客户端使用权限、业务目录管理权限；
kitty： 客户端使用权限；
sysadmin： 受管引擎实例管理权限；
